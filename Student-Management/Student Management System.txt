Basic Idea of the Three Layers:
The frontend (HTML, CSS, and JavaScript) collects user input and sends it to the Flask backend using the Fetch API. The Flask backend processes the request, interacts with the MySQL database to store or retrieve data, and returns the result as JSON. The frontend then uses the JSON response to update the webpage dynamically without refreshing it.
 
3-Layers of this Task in Brief:
The frontend was developed using HTML, CSS, and JavaScript. It provides the user interface where users can add, edit, delete, and search student records. Whenever a user performs an action, JavaScript uses the Fetch API to send HTTP requests to the Flask backend.

The backend was developed using Python and Flask. Flask receives the requests from the frontend, validates the input data, executes the required business logic, and communicates with the MySQL database using parameterized SQL queries through the mysql-connector-python library.

The MySQL database stores all student information permanently. After completing the requested operation, the backend returns a JSON response to the frontend, and JavaScript updates the webpage dynamically without requiring a page refresh.

In simple terms:

Frontend → Flask Backend → MySQL Database → Flask Backend → Frontend

2.Project Implementation in Details:
The project was implemented using a three-layer architecture consisting of a frontend (HTML, CSS, JavaScript), a backend (Flask), and a MySQL database. I developed RESTful APIs to perform complete CRUD operations and integrated them with the frontend using the Fetch API for seamless data exchange. Additionally, I implemented dashboard statistics, backend search functionality, responsive UI design, and dynamic table updates to provide a smooth and user-friendly experience.

As i Already mentioned the most challenging part of this project was establishing a successful connection between Flask and the MySQL database. I solved this by verifying the MySQL username, password, and host address in MySQL Workbench. I tested the connection separately using a simple Python script before integrating it into the Flask application.

Another challenge was implementing the Edit (PUT) and Delete (DELETE) operations. I had to understand how HTTP methods work and how JavaScript communicates with the backend using the Fetch API. After understanding the request-response flow and debugging my JavaScript functions, I successfully implemented the CRUD operations.

Finally, I faced some frontend alignment issues where form elements, buttons, and the table layout were not properly aligned. I corrected these by restructuring the HTML and improving the CSS using Grid layout and responsive styling even though our Mam told not to concentrate more styling looks I done the Basic level to make it aligned properly.
