1. What is password hashing and why is storing plain text passwords dangerous?
From my Understanding the Password hashing is the process of converting a password into an irreversible encrypted string using a hashing algorithm such as Bcrypt. Like we are converting the binary file just a refernce i am telling. During user registration, the original password is hashed before it is stored in the database. When a user logs in, Flask-Bcrypt hashes the entered password internally and compares it with the stored hash. If both match, authentication is successful.
Storing passwords in plain text is dangerous because anyone with access to the database can immediately view users' passwords. If the database is compromised, attackers can use those passwords to access other accounts where users may have reused the same password. Hashing protects user credentials by ensuring that the original password cannot be recovered from the stored value.

2. What is a session and how does Flask use it?
After a user successfully logs in, a session is used to save their identification. Flask saves critical user data, such the user ID, username, and role, in a secure session rather than requiring the user to submit their credentials on each request.

Flask determines if the session is active when the user hits protected routes, such as the dashboard or profile. Access is allowed if the session is legitimate. If not, the user is sent to the login page and receives an unlawful response from the program. When you log out, the session is cleared, making it impossible to view protected pages again.

3. What happens on your backend when a wrong password is entered?
The backend looks for the supplied username in the database when it receives a login request. Flask-Bcrypt uses check_password_hash() to compare the entered password with the stored hashed password if the username is present.

Flask does not establish a session if the passwords do not match; instead, it sends an HTTP 401 Unauthorized response with the message "Invalid Username or Password." This keeps stored passwords safe and guarantees that only users with the right credentials can access protected services.

5.Project Implementation:
This project is a secure User Login and Authentication System developed using Python, Flask, MySQL, HTML, CSS, and JavaScript. The application allows users to register by creating a new account, where passwords are securely hashed using Flask-Bcrypt before being stored in the MySQL database. Registered users can log in using their credentials, and upon successful authentication, a Flask session is created to provide access to protected pages such as the dashboard and profile. The dashboard displays user-specific information, including the username, role, email, and account creation date. A logout feature securely clears the active session, ensuring that protected resources cannot be accessed without authentication. The frontend communicates with the backend using the Fetch API, providing a smooth user experience without page reloads.
